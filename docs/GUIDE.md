# Auto Report Guide

Task-oriented reference for the productivity report system.

## Workflows

### Add a new JCR paper

1. Add an `@Article` entry to [`database/myproducts.bib`](../database/myproducts.bib)
2. Set `note = {jcr}`, `year`, `month`, `if`, and `doi`
3. Run `python -m reports generate full-cv --lang es`
4. Compile: `cd reports/full_cv/templates && pdflatex cv_jivg_es_full.tex`

### Report for a specific period

```sh
python -m reports generate full-cv --lang es --from 2024-01-01 --to 2025-12-31
```

Or edit `date_from` / `date_to` in [`config.yaml`](../config.yaml).

### Update website publications

```sh
python -m reports generate web-page
```

Upload or sync `reports/web_page/html/publications.html`.

### Prepare EDI submission

1. Set `edi_output` and `edi_probatorios` in `config.yaml`
2. Run `python -m reports generate edi --from 2024-01-01 --to 2025-12-31`
3. Review printed point totals and output folder

### Update personal bio or statement

Edit these files **by hand** (not regenerated by the CLI):

| File | Content |
|------|---------|
| `reports/full_cv/text/personal_statement_es.tex` | Personal statement (ES) |
| `reports/full_cv/text/personal_statement_en.tex` | Personal statement (EN) |
| `reports/full_cv/text/short_bio_es.tex` | Short biography |
| `reports/full_cv/text/impact_statement.tex` | Scientific contribution |
| `reports/full_cv/text/certifications.tex` | Certifications |
| `reports/full_cv/text/reviewer.tex` | Editorial profile |
| `reports/full_cv/text/awards_es.tex` | Awards |
| `reports/full_cv/text/projects.tex` | Projects |

Then recompile the LaTeX template only.

## Static vs generated files

Files in `reports/*/text/` and `reports/web_page/html/`:

### Generated (overwritten by CLI)

`jcr_journals`, `conacyt_journals`, `proceedings`, `books`, `otros`, `master_students`, `phd_students`, `teaching`, `developments`, `talks`, `workshop`, `divulgacion`, `eventos`, `entrevistas`, `preprints`, `products.png/pdf`, HTML equivalents.

### Hand-edited (never overwritten)

`personal_statement_*`, `short_bio_es`, `impact_statement`, `certifications`, `reviewer`, `awards_es`, `projects`, `encabezado.html`, `my_thesis.html`.

## Architecture

The system follows a template + fragments pattern:

1. **Database** — BibTeX and CSV source files
2. **Python** — load, filter by date, categorize, write fragments
3. **Templates** — LaTeX master documents with `\input{../text/section}` 
4. **PDF** — `pdflatex` on the template

```
myproducts.bib  →  categorize  →  writers.py  →  text/jcr_journals.tex
                                                      ↓
                              templates/cv_jivg_es_full.tex  →  PDF
```

## Troubleshooting

### Section is empty but I have publications

Check that `year` and `month` fall within your `--from` / `--to` range. Run `python -m reports stats` to see counts.

### "Missing key 'month'" warning

Add `month = {MM}` to the BibTeX entry. Entries without required keys are skipped.

### LaTeX: `\input` file not found

Run the generator before compiling: `python -m reports generate full-cv --lang es`

### Special characters break LaTeX

Titles with `&`, `%`, `_` may need LaTeX escaping. The writers pass text through as-is from BibTeX.

### divulgacion section missing (legacy notebooks)

Older notebooks had a bug gating divulgación on JCR count. The CLI uses the correct `divul` category count.

## Notebooks

Jupyter notebooks in each report folder (`text_generator.ipynb`, etc.) were the original interface. They remain for chart debugging and exploration. For reproducible output, use:

```sh
python -m reports generate <report-name>
```
